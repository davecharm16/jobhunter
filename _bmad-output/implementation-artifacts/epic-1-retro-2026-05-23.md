## 2. What Went Well

### Successes carried by evidence

1. **Tight scope discipline across all five stories.** Every story's Dev Notes enumerated explicit "what this story does NOT do" guardrails with story-number forward references. Every implementation respected them. No epic creep.
2. **Test seam strategy was correct on day one.** `run_tailoring(..., llm_tailor=llm_client.tailor)` plus a stub-injection helper (`_isolated_cli_env_with_fake_llm`) meant zero tests ever hit `api.anthropic.com`. Mock cost is wrong, determinism is wrong, CI signal is wrong вЂ” we didn't pay any of that.
3. **Adversarial review caught real defects every story.** Across 5 reviews + 1 second-pass review on 1.5, the auto-fix lane caught: a custom dotenv parser violating an explicit guardrail (1.2); a CLI subprocess test polluted by the dev's real `.env` (1.2); a missing `UnicodeDecodeError` handler that crashed AC7 (1.4); a missing `OSError` catch on artifact-write (1.5 H1); a non-finite `inf` accepted as `LLM_CALL_TIMEOUT_SECONDS` (1.5 M1); a top-level ledger type that wasn't validated and would crash with `AttributeError` (1.5 second-pass H1); and вЂ” most importantly вЂ” a missing `response.usage` validation that would have silently recorded $0 per call and **defeated the monthly cap** (1.5 second-pass H2). Without adversarial review these would have shipped.
4. **Atomic-write pattern as load-bearing infrastructure.** Build into `<slug>.tmp/`, then `os.replace` to the final `<slug>/`. Same pattern used for the cost ledger. Crash mid-write cannot leave half-written artifacts or a half-written ledger. This was the right primitive to invest in early.
5. **`Decimal` for money, never `float`.** Pricing constants, ledger totals, and cap arithmetic all run through `Decimal`. Ledger serialization uses quoted strings so round-trip drift is zero. Cost arithmetic is now structurally trustworthy.
6. **Tool-use over free-form JSON for structured output.** `tool_choice={"type": "tool", "name": "emit_tailored_artifacts"}` forces the model into the two-field schema. AC6 has a structural reliability mechanism, not a prose-parsing one.
7. **TTY detection prevents interactive hang.** Story 1.4's `sys.stdin.isatty()` short-circuit means `jobhunter paste` in a real terminal exits 2 immediately instead of blocking on `stdin.read()`. The in-process test uses `pytest.fail` inside a stub `read()` to make the AC4 contract load-bearing.
8. **Helper extraction at the right moment.** Stories 1.2/1.3 duplicated `_isolated_cli_env` across files. Story 1.4 review extracted to `tests/integration/_cli_helpers.py`. Story 1.5 extended with `_isolated_cli_env_with_fake_llm` against a clean home. The refactor paid for itself within one story.
9. **DECISIONS.md treated as additive, not mutable.** В§1 (runtime), В§2 (schema), В§3 (revisit triggers), В§4 (LLM provider) were written once with rationale + rejected alternative + revisit conditions. No silent history rewrites. Future-you can audit why.
10. **Pytest was ratified, not relitigated.** Story 1.1 adopted it ahead of plan; Story 1.2 ratified rather than reopen the decision. The cost of the early scope expansion was repaid immediately.

## 3. What Was Hard / Recurring Patterns

### Friction points worth naming

1. **Every story needed adversarial-review auto-fixes before reaching `done`.** Five stories, five review passes (Story 1.5 needed two passes). The pattern is consistent enough to call out: dev-story produces work that ships, but only after review catches real bugs. Read positively: review is doing its job. Read negatively: dev-story is under-checking for a small recurring set of issues (missing exception classes, doc/status drift, helper duplication, message-wording collisions).
2. **Boundary-message renames cascaded into adjacent-story tests.** Story 1.4 had to rename two Story-1.2-era boundary tests. Story 1.5 had to rename + rewrite two Story-1.4-era happy-path tests. Each story's "tests for prior boundary" became a chore. Acceptable trade-off given the walking-skeleton pace, but worth being conscious of for Epic 2 where the success message changes shape again (metadata sidecar, drift verdicts, etc.).
3. **Doc/status drift caught in review, not by dev.** Story 1.3 review caught README Status still claiming only Story 1.1 done; Story 1.5 had `DECISIONS.md` footer date drift. Cheap to fix but it surfaced in review consistently. A "before you say done, grep README for last story number" habit would catch this earlier.
4. **PyPI / sandbox flakiness produced silent test skips.** Stories 1.2вЂ“1.5 all ran in a sandbox where DNS to PyPI was blocked. Two `python-dotenv`-conditional tests have been `skipped` since Story 1.2 and never re-enabled. Story 1.5 added the same `pytest.importorskip("anthropic")` pattern вЂ” correct given the constraint, but the skipped count is now load-bearing as a known-state signal. Re-enabling needs an explicit "test the sandbox case" pass.
5. **String-substring collisions in CLI copy.** Story 1.5's `NO_AUTO_SUBMIT_STATEMENT` contained `OnlineJobs.ph`, which collided with the AC8 source-grep test for `onlinejobs.ph`. The fix (reword to `OnlineJobs PH`) is fine, but it surfaced a class of issue: when a test asserts a substring is *absent* from source, every piece of human-facing copy needs to be aware of the negative space.
6. **`record_call` can leave a stray `.cost-ledger.json.tmp`** if `os.replace` fails mid-rename. Atomic-rename atomicity is preserved (the real ledger is intact), but the temp file persists until the next call overwrites it. Accepted as walking-skeleton; deferred. Watch this when Epic 7 concurrency arrives.
7. **Per-month-entry ledger validation deferred.** The top-level `dict` check landed in Story 1.5 second-pass H1, but a per-entry shape like `{"2026-05": "not-an-object"}` still raises `AttributeError` in `current_month_total_usd`. Cosmetic for Epic 1 (the ledger is only ever written by `record_call`); becomes load-bearing in Story 2.12 when `jobhunter stats` reads it.
8. **Story 1.1 quietly expanded scope (pytest adoption) ahead of plan.** Completion Notes confessed honestly. It worked out because pytest was the right call, but the precedent worth naming: when a dev sees a near-term need and the story scope is the only block, the dev should either (a) update the story plan first or (b) flag the expansion explicitly in completion notes (which 1.1 did, eventually). Don't normalize quiet drift.

## 4. Key Insights & Lessons (apply to Epic 2)

1. **Adversarial review caught a wallet-draining cap bypass in Story 1.5 (`response.usage` missing в†’ $0 recorded в†’ cap never trips).** This is the single highest-value find of the epic. Keep adversarial review on the critical path for every story; treat the "safety invariant" surface (cap, no-auto-submit, no JD persistence, no canonical-CV mutation) as the highest-leverage place for the reviewer's attention.
2. **Mock the LLM at the function seam, not the SDK.** `llm_tailor=` injection into `run_tailoring()` was clean. Subprocess-level mocking via a swapped stub `llm_client.py` was also clean. Both kept tests fast and deterministic. Epic 2 adds at least two more LLM calls (Story 2.3 JD parser; Story 2.7 Upwork proposal); the seam pattern scales вЂ” add `parse_jd=` and `generate_proposal=` injection points the same way.
3. **The temp-rename pattern is the right primitive for any multi-file artifact write.** Reuse for `out/<slug>/metadata.json` (Story 2.10) and any future package-of-files write. Resist the urge to write `.json` first and `.md` second "because it's small".
4. **Configurable thresholds belong in `config.yaml` (Story 2.2), not in `.env`.** `MONTHLY_SPEND_CAP_USD` ended up in `.env` because Story 1.2's AC said so вЂ” but it's a tunable, not a secret. Story 2.2 should move it to `config.yaml` and update `runtime_config.py` accordingly. Same with the `LLM_CALL_TIMEOUT_SECONDS` override added in Story 1.5. And the pricing constants in `llm_client.py`. And `MODEL_NAME`.
5. **Source-grep tests are cheap and load-bearing.** The "no `requests`/`click`/`upwork.com`/`linkedin.com` in `src/jobhunter/`" test caught the `OnlineJobs.ph` collision and protects FR11 / FR44 structurally. Extend the same pattern to forbid `selenium`, `playwright`, `bs4` ahead of Epic 7.
6. **`pytest.importorskip` is the right pattern for SDK-conditional unit tests.** Don't let an SDK-less venv produce a screen of import errors; produce one skip with a clear message. Story 1.5 used it for `anthropic`; reuse for whatever Epic 2 adds (likely `pyyaml` for `config.yaml`, possibly `fastapi`/`starlette` for the `POST /ingest` endpoint).
7. **Walking-skeleton scope worked because each story had explicit "what NOT to do" guardrails.** Continue this pattern. Epic 2 is broader (12 stories vs 5), so the guardrails matter more, not less.

### Likely architectural shifts in Epic 2

1. **`config.yaml` loader becomes a peer of `runtime_config.py`.** Pick a YAML library (recommend `pyyaml` вЂ” already common, no maintenance flag вЂ” but verify against current PyPI at story creation). Establish a `LoadedConfig` dataclass parallel to `RuntimeConfig`.
2. **Pipeline becomes multi-step + multi-LLM-call.** Story 1.5's `run_tailoring()` is a single-call orchestrator. Story 2.3+ need a real pipeline: `parse в†’ classify в†’ select_artifacts в†’ tailor_each в†’ write_metadata`. Plan for a `pipeline.py` module that holds the chain, with each step swappable for tests.
3. **Per-application metadata sidecar replaces stderr-only success path.** The current "print to stderr, exit 0" success message becomes additionally a structured JSON write. Update test helpers to read this rather than parse stderr.
4. **`POST /ingest` endpoint introduces the first web framework dependency.** Story 2.11 вЂ” likely FastAPI or Starlette. This is a **significant architecturally-relevant addition**: it adds a new runtime entry point (server), a new test surface (HTTP), and the first concurrent-write risk to the ledger. Treat the endpoint and the CLI as two front ends over **one pipeline function**, not two pipelines.
5. **Prompt templates become files, not module-level strings.** Story 2.9 moves the inline `SYSTEM_PROMPT` in `llm_client.py` to `prompts/cv.v1.md` (and friends). Plan the loader and the version-extraction shape.
